# Автоматическая проверка развернутых ответов в LMS

## Концепция
Студенты пишут ответы на открытые вопросы (не тесты). Система должна проанализировать ответ, сравнить его с эталоном или критериями оценки, выставить балл и аргументировать его.

## Научная / Инженерная новизна
Исследование точности «микромоделей» (SLM — Small Language Models) в задачах классификации и семантического сравнения текстовых эмбеддингов (смысловых векторов). Оптимизация инференса (скорости работы) локальной модели, чтобы она не требовала терабайтных мощностей.

## Архитектура и этапы реализации:
1. Пайплайн RAG: Формирование векторной базы данных на основе лекций, методичек и эталонных ответов (с использованием FAISS, ChromaDB или Qdrant).
2. Гибридный скоринг: Сначала ответ проверяется по косинусному сходству векторов (эмбеддингов) с эталоном, затем передается в модель.
3. Локальный LLM-агент: Развертывание квантованной модели (например, Llama-3-8B-Instruct-GGUF или Mistral-7B) на локальном сервере с использованием llama.cpp или vLLM.
4. API Интеграция: Написание FastAPI-сервиса, который принимает ID студента, ID вопроса и текст ответа из LMS, а возвращает JSON с оценкой и рецензией.

## Стек технологий
Python, vLLM, LangChain / LlamaIndex, ChromaDB, FastAPI.

## Критерии успеха (KPI)
Метрика BERTScore или ROUGE между оценкой ИИ и
оценкой эксперта-преподавателя должна коррелировать на уровне не менее 85%
(на тестовой выборке из 100+ ответов).
